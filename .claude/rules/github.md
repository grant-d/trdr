# Github

## Workflow

1. Feature branch from `main`
2. Add/update tests + docs
3. Ensure no type errors (TS or Python)
4. PR with clear description

## PRs

- Push branch first when creating PR
- NEVER push or commit without explicit user request

## Commands

- Use simple commands when in repo root: `git status` not `git -C /full/path status`

## Contributing

- All changes require review
- Maintain/improve coverage
- Breaking changes: discuss first
- Conventional commit format

## Versioning

- Semver (MAJOR.MINOR.PATCH)
- Update CHANGELOG.md
- Tag releases
